Lately, I've seen many people starting to approach Antigravity Coding - that is, letting the AI agent write code, run it, and fix errors almost without manual intervention.

Sounds very appealing. But in reality, most beginners make the same mistakes — causing AI to work inefficiently, even breaking the code instead of fixing it.

Error 1: The prompt is too vague
"Make my app work" — AI is not divine. It needs to know what the app is, where the error is, and what the context is.

How to fix: provide enough context — describe the feature, paste the specific error message, specify the tech stack being used.

Error 2: Letting the agent run uncontrolled
Many of you turn on the agent and then go do something else, only to come back and find that the AI has deleted several files and rewritten all the logic in a completely different way.

Fix method: use Plan Mode first — have the AI make a plan, you review it, and then let it run. Control each important step.

Error 3: No "Memory"
The agent does not have memory of your project. Each session starts from scratch.

Fix method: set up the Rules file from the beginning — clearly outline the project structure, coding rules, and the libraries being used. This is the handover document for the AI every time it "wakes up."

Error 4: Trusting AI 100% without double-checking AI can confidently write code that completely violates business logic without reporting any errors.

Fix method: always review what AI has changed thru diff or git. Don't skip this step even if the code runs. If possible, you should write documentation for your project.

Error 5: Stuffing everything into one prompt "Write me the entire sales management system, with login, shopping cart, payment, reports..." — the agent will start, then get lost halfway, either missing something or making mistakes.

Fix method: break down the task. Each time, assign only one specific feature, and only move on to the next one after it is confirmed. Use Skill and Workflow to reuse prompts.
